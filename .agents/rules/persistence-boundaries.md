# Persistence Boundaries

本文件定义 `.agents/` 中 Rules、Skills、Memory 和 Changelog 的职责边界。

## Rules

Rules 是项目级稳定约束，适合记录所有后续 Agent 都应遵守的格式规范、流程强制、项目约定和安全边界。

适合写入 Rules：

- changelog 格式、日期格式、分类枚举等格式规范。
- 修改前必须查看 git 状态、不得覆盖无关用户改动等安全约束。
- 构建、主题、内容变更后的验证要求。
- 本项目特有且长期有效的 Agent 操作规则。

不适合写入 Rules：

- 可从代码或配置直接推导的信息。
- 机器特有绝对路径、账号、代理或私有环境变量。
- 用户个人偏好。
- 当前会话进度、短期计划或临时项目状态。

## Skills

Skills 是任务类型操作手册。它们不重复项目约束，而是描述一类任务应该如何执行。

适合写入 Skills：

- 审计日志更新流程。
- 新增页面、修改主题、调整构建部署等可复用流程。
- 需要 checklist 的复杂任务。

## Memory

真实 Memory 不应存储在本仓库中。本仓库只保留 Memory 机制说明，避免把私人或机器特有信息误提交。

如果 Agent 平台支持私有 Memory，可把以下内容写入平台私有 Memory：

- 用户长期偏好。
- 本机路径、代理、工具安装位置等环境事实。
- 外部系统入口或短期项目目标。

## Changelog

Changelog 是 append-only 的事实时间线，只记录大型或重要改动。它解释项目为何演化到当前状态，但不替代源码、配置和 Git 历史。
